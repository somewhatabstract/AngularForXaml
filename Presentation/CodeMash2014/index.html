Spent 12 years doing Windows. Mastered the domain. I've been doing web development for almost a whole year.

When I changed jobs, I felt like I showed up to an event with the wrong kit. Lots to learn. Where to get started. Time to find similarities.

What I knew of web development was a handful of JAvaScript, some HTML and a bit of CSS, and Silverlight.

Silverlight (originally code-named WPF Everywhere) is a .NET runtime that runs in and out of the browser on both PC and Mac. It requires a plugin but allows you to write rich web apps using C#/VB and XAML.

If into acronyms, it's a MVW framework (Model-View-Widget). Unlike jQuery, a utility library, Angular is a platform on which to build client-side applications. Angular is to jQuery as the Silverlight runtime is to the System.Collections namespace.

  • Terms like filters, modules, factories, services, providers, and directives everywhere.
  • Javascript/HTML/CSS
  • But I didn't know Angular and I only had a passing acquaintance with JS, HTML and CSS
  • I knew Silverlight, C# and XAML.

We seemed to be talking about the same things under different guises. I needed to find the link. So, I started to look for concepts that matched up.

Data Binding


<StackPanel x:Name="LayoutRoot">
    <TextBox x:Name="textStuff" />
    <TextBlock Text="{Binding ElementName=textStuff, Path=Text}" />
</StackPanel>
                    
<div ng-app>
    <input ng-model="textStuff" />
    <div>Hello {{textStuff}}</div>
</div>

This is a bit of familiar Silverlight, a textbox bound to a textblock. Simple data binding.

Let's see that in Angular.

<div ng-app>
    <input ng-model="textStuff" />
    <div>Hello {{textStuff}}</div>
</div>
Hello {{textStuff}}

Breaking it down. The ng-model directive is telling the textbox what value it is bound to. To present that value, we use the double braces syntax.

Two way binding works too. Just add another input.

<div ng-app>
    <input ng-model="textStuff" />
    <div>Hello {{textStuff}}</div>
    <input ng-model="textStuff" />
</div>

<StackPanel x:Name="LayoutRoot">
    <TextBox x:Name="textStuff" />
    <TextBlock Text="{Binding ElementName=textStuff, Path=Text}" />
    <TextBox Text="{Binding ElementName=textStuff, Path=Text}" />
</StackPanel>

Comparing this with Silverlight, you can see a difference. In Silverlight as it is here, we are binding to one of the two textboxes. In Angular, the bindings are two way and bind to a model.

Usually, we would bind to a value in a model specified via the data context. Angular creates this model for us.

But, we can also have the concept of a data context in Angular to get finer control over our model.

In fact, you probably noticed there's a default value here; "World" - that requires code behind.

Just like Silverlight, the code-behind has to be associated with the presentation layer.

In Silverlight, this is done by specifying the XAP that implements the app and the x:Class attribute that indicates the class that contains the code.

<div ng-app='myApp' ng-controller='myController'>
    ...
var myApp = angular.module('myApp', []);
myApp.controller('myController', function ($scope) {
    ...

In Angular, the ng-app directive needs a name to specify the angular module that contains the code and there has to be an ng-controller to specify the function that controls an element. 

<div ng-app='myApp' ng-controller='myController'>
    <input ng-model="textStuff" />
    <div>Hello {{textStuff}}</div>
    <input ng-model="textStuff" />
</div>
(function(){
    var myApp = angular.module('myApp', []);
    myApp.controller('myController', function ($scope) {
        $scope.textStuff = 'World';
    });
})();

Here's the full Angular app for our earlier data binding demo with two textboxes.

Note the [] in the module declaration. This is where additional modules can be injected so that they can be referenced, much like adding references to a Silverlight project.

The $scope value of the controller is injected by Angular and is the model that bindings reference. Other things can be injected here too, such as things from the referenced modules.

More on that later. Back to binding - this time collections.

In Silverlight, some controls have an ItemsSource property for specifying a collection that can be used to generate items.

ng-repeat performs this task in Angular.

<p ng-repeat="name in names">{{name}}</p>
$scope.names = [ 'Jim', 'John', 'Bob'];

Inside of our angular app, we add an ng-repeat directive to the element that will repeat (our ItemTemplate, if you will).
